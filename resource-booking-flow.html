Customer Journey (Full Path)

1

Reserve page selection

Customer picks one or more 1-hour resource slots on the reserve page. Selections are stored temporarily and the customer continues to checkout.

2

Checkout review and prerequisites

Checkout loads selected items, applies pricing/tax context, loads credits and passes, validates waiver acceptance, and collects optional guest details.

3

Payment initialization

The app requests a secure payment session and prepares the payment UI for card checkout.

4

Payment execution path

Customer completes one payment strategy: new card, saved card, credits-only, pass-only, or mixed payment (credits/pass plus card remainder).

5

Order fulfillment

Order fulfillment creates the booking, marks selected time slots as reserved, writes payment records, updates credits/passes, and applies promotion usage when applicable.

6

Confirmation and communication

Checkout sends booking confirmation, opens the confirmation page, and the user can then review records in My Bookings.

7

Door access instruction (Phase 1)

Instruction email uses a secure access link. The access gate checks status and timing, then either shows a wait message, a blocked message, or continues to the legacy door-code page.

8

SimplyBook migration bridge

Backend cron scans recent booked/course-reserved slots and posts Zapier payloads for parallel sync during migration, without changing booking controller logic.

Payment and Fulfillment Variants

Payment options

  • Full Stripe (new card)
  • Saved card
  • Credits-only
  • Pass-only
  • Mixed credits + Stripe remainder
  • Mixed pass + Stripe remainder

Fulfillment outputs

  • Booking record is created and confirmed.
  • Each selected slot is persisted/updated as booked occupancy.
  • Payment record is written and linked to booking.
  • Credits/passes/coupon counters are updated when used.
  • Success page and My Bookings reflect the completed transaction.
Verification target used by planned E2E: reserve page → checkout → confirmation page → My Bookings.

Phase 1 Add-ons Around Booking

Door access gating

  • Public entry point: secure door access link from email.
  • Access outcomes: wait message, blocked message, or continue to code page.
  • Redirect target is legacy Google Script page.
  • Access window opens 10 minutes before scheduled start.

SimplyBook parallel sync

  • Background sync task runs on a recurring schedule.
  • Sends booking reference, service, court, time range, price, and client details to the migration bridge.
  • Includes queue tracking, de-duplication, retry backoff, and dead-letter handling.
  • Default cadence is every minute with configurable timezone support.
