Backend and BFF testing overview

This page summarizes the current automated test coverage across the Open24 backend and the BFF layer, including unit tests, API/service tests, and end-to-end smoke checks.

Backend: 19 test files / 210 tests BFF: 30 test files / unit + API coverage E2E smoke journeys included

Coverage Snapshot

Backend

210 tests

Coverage focuses on core services, middleware guards, booking and enrollment logic, pricing and credits, promotion, webhook handling, and email lifecycle helpers.

BFF

263+ tests

Coverage focuses on server route behavior, request validation, proxy reliability, checkout and payment flows, door access gating, and shared client/server utilities.

Test counts reflect the current engineering test inventories and can change as the codebase evolves.

BFF Test Areas

  • Server utility behavior such as error handling, logging redaction, URL construction, and request helpers.
  • Route-level validation and forwarding for auth, bookings, enrollments, credits, legal content, and payment actions.
  • Door access gate decisions and redirect behavior for wait, blocked, and allowed outcomes.
  • Middleware protections such as rate limiting and guest route safeguards.
  • Client utility coverage for pricing computation, timezone handling, date/phone formatting, and status display helpers.
  • Browser smoke tests for guest access control and core checkout entry scenarios.

Backend Test Areas

  • Token and middleware safeguards including signature validation, JWT parsing, and internal secret checks.
  • Booking and course enrollment support logic including helper functions, fulfillment rules, and edge-case handling.
  • Credit, pricing, holiday, and promotion logic including priority resolution and accounting accuracy.
  • Gift card creation and redemption logic including uniqueness, limits, and status constraints.
  • Email rendering and lifecycle scheduling helpers including time offset and timezone-sensitive calculations.
  • Stripe checkout and webhook controller behavior for payment and post-payment processing.

Test Levels

Level Goal Typical Coverage
Unit Verify isolated logic quickly and deterministically. Formatting helpers, pricing math, token helpers, pure utility functions.
Service / API Verify request validation, orchestration, and persistence-facing behavior. Booking, enrollment, payment flows, email dispatch logic, middleware policies.
E2E Smoke Validate top-level customer journeys and route protections. Login gating, checkout entry handling, and critical page-level navigation paths.

Quality Gates and Current Gaps

Quality Gates

  • Code quality checks include linting, type checking, and automated tests.
  • Backend and BFF both use Vitest for fast automated test execution.
  • BFF also maintains Playwright smoke coverage for browser-level safeguards.

Current Gaps

  • E2E smoke coverage is intentionally lightweight and should be expanded for deeper scenarios.
  • Several complex controller paths still rely more on integration behavior than direct unit-level coverage.
  • Role-specific and high-variance UI workflows need broader end-to-end assertions over time.
Source: internal backend and BFF testing inventories.
